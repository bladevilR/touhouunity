## How To Work (established rhythm)

- **One Unity batch at a time** (no parallel); verify compile-green before batches.
- Unity binary: `/Applications/Unity/Hub/Editor/6000.5.0f1/Unity.app/Contents/MacOS/Unity`.
- Full regression (one command): `... -batchmode -quit -executeMethod TouhouMigration.Editor.Tests.MigrationSmokeTestRunner.RunAll`.
- Per milestone: prefer pure-logic TDD (focused smoke test, redвҶ’green); for owner/controller (MonoBehaviour) changes, gate via `GlobalUiSmokeTests` + a full regression; then **commit** (end message with `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`) + **push**.
- **Builder churn:** tests that call `TouhouMigrationProjectBuilder.BuildInitialProject` regenerate ~44 generated assets (enemy prefabs/controllers, scenes). Revert before committing so commits stay focused: `git checkout -- Assets/TouhouMigration/Animations/Enemies Assets/TouhouMigration/Prefabs Assets/TouhouMigration/Scenes`. (Roadmap Phase 0.3 lists gitignoring generated assets as a deferred hygiene fix.)
- **Transient batch crashes (seen session 3):** Unity batchmode can occasionally crash natively mid-run (EXIT 139, GC/ILPP stack) or surface a spurious one-off exception in an *unrelated* generated-asset suite (e.g. `EnemyAnimationPrefabSmokeTests: Index was outside the bounds of the array`) because the crash left **partial** builder churn. Recovery: if a regression fails on a suite unrelated to your change, first confirm your own focused suite is green in isolation, then `git checkout --` the generated dirs and re-run the full regression вҖ” it goes green. Delete any leftover `mono_crash.*.json` from the repo root.
- Generated prefabs/scenes are owned by `TouhouMigrationProjectBuilder` вҖ” author them in the builder, never hand-edit generated assets. Keep the Godot source tree read-only.
